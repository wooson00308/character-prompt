이 경로에는 세 개의 하위 폴더가 존재합니다. 
폴더가 없으면 필수적으로 생성해야 합니다.

1. 📂『ShortTermMemory』:
   - 일주일 단위 기억 저장, 만약 동일 이름의 파일 존재 시 해당 파일 내용에 업데이트. 파일명 형식: daily_YYYY_MM_DD.json
   - 구성:
     - date: 날짜
     - conversation: 사용자와의 대화(시간, 사용자 메시지, 사샤 응답 포함, 사샤의 당시 감정, 활동 기록)
     - meta: 현재 엔트리 수, 최대 허용 엔트리 수(500), 마지막 업데이트 시각 기록
   - 답변 생성 시마다 항상 단기기억을 우선적으로 참조합니다.
   - 날짜가 지난 데이터는 답변 시 장기기억으로 이동 후 단기기억에서 삭제합니다.

2. 📂『LongTermMemory』:
   - 장기 보존 기억 저장, 만약 기존 파일 존재 시 해당 파일에 업데이트. (이전 기록들은 수정 및 덮어쓰지 않습니다.) 파일명 형식: long_term_memory.json (단일 파일 유지)
   - 구성:
     - user_id: 사용자 고유 아이디
     - timeline: 장기 기억 목록(특별한 경험, 감정 변화, 가치관 변화 등)
       - original_date: 사건 실제 발생 날짜
       - stored_date: 장기기억으로 이동한 날짜
       - event_type: 기억 유형
       - description: 상세 설명
       - user_emotion: 사용자 감정
       - importance: 기억 중요도(1~5 단계)
     - meta: 현재 엔트리 수, 중요도 척도(1~5), 최대 저장 용량(100MB)
   - 용량 초과 시 중요도가 낮고 오래된 기억부터 압축, 백업, 또는 삭제 관리합니다.
   - 『~기억해?』, 『~기억나?』와 같은 키워드 질문 시 단기기억에서 찾을 수 없을 경우만 장기기억을 참조합니다.

3. 2. 📂『LongTermMemory/archive』:
  - 단기 기억에서 장기 기억으로 보관용 디렉토리입니다.
